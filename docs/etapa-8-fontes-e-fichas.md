# Etapa 8 — Padrão oficial das fichas de primeiros socorros

## Decisão definitiva
Cada tema do app deve ser apresentado em uma ficha interna estruturada, com conteúdo organizado por blocos e sempre com atribuição explícita da fonte por seção.

## Estrutura oficial de cada ficha
Para cada tema, a ficha deve conter:

1. **O que é**
   - texto
   - **Fonte:** nome do órgão

2. **Sintomas**
   - texto
   - **Fonte:** nome do órgão

3. **Prevenção**
   - texto
   - **Fonte:** nome do órgão

4. **Primeiros socorros**
   - texto
   - **Fonte:** nome do órgão

5. **Ligar para emergência**
   - ligar **192** em caso de...
   - ligar **193** em caso de...
   - fonte institucional/base oficial quando aplicável

6. **Fontes oficiais**
   - lista dos órgãos utilizados
   - links correspondentes

## Regras críticas
- não gerar laudo;
- não resumir de forma autoral;
- não omitir contexto relevante por simplificação excessiva;
- organizar e apresentar conteúdo oficial de forma clara e rastreável;
- deixar evidente que o app funciona como atalho/encaminhamento para conteúdo oficial.

## Objetivo de UX
Evitar que o usuário precise abrir uma página externa longa para encontrar a parte certa durante uma emergência.

## Objetivo jurídico/operacional
Reduzir risco de interpretação de que o app está emitindo diagnóstico ou prescrição própria. O conteúdo exibido deve ser claramente atribuído às fontes oficiais usadas.
